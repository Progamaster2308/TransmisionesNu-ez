RESPALDO Y REVERSA DEL ENVIO DE CORREO

Proveedor nuevo:
- Web3Forms se usa primero si existe VITE_WEB3FORMS_ACCESS_KEY.
- Si no hay key o falla Web3Forms, la app cae al FormSubmit anterior.

Archivo principal:
- src/app/providers/orderMailer.js

Archivo de respaldo FormSubmit:
- src/app/providers/orderMailer.formsubmit-backup.js

Como volver al sistema anterior:
1. Copia el contenido de:
   src/app/providers/orderMailer.formsubmit-backup.js
2. Pegalo reemplazando todo el contenido de:
   src/app/providers/orderMailer.js
3. Quita VITE_WEB3FORMS_ACCESS_KEY de Netlify si ya lo agregaste.
4. Ejecuta:
   npm run lint
   npm run build
5. Haz commit y push.

Nota:
- Si no configuras VITE_WEB3FORMS_ACCESS_KEY, el codigo actual sigue usando FormSubmit como respaldo.
- Para activar Web3Forms en Netlify, crea una access key en Web3Forms y agregala en:
  Netlify > Site configuration > Environment variables
  Nombre: VITE_WEB3FORMS_ACCESS_KEY
  Valor: tu access key
